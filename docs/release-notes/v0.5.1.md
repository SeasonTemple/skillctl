## Implementation notes

- **No production logic changed.** All targeted paths were
  correct-but-untested; none surfaced a root-cause bug (no in-scope fix
  was needed ‚Äî the v0.2.0 fix-at-root precedent did not trigger).
- **Discovered at implementation:** `updateMulti` **and**
  `uninstallMulti` both lack an `allowNoCli` parameter (only
  `installMulti` has it). The uniform gate-bypass for `assertCliPresent`
  is therefore a `PATH`-stub (`whichSync` only checks `isFile()`), not
  `allowNoCli` ‚Äî plan-003's and the plan's framing implied `allowNoCli`
  for `uninstallMulti`, which is inaccurate; the tests use the correct
  PATH-stub mechanism.
- The product-literal guards (now `skillctl|netops|nexel`) constrain new
  test strings; the `startSpinner` default-label test asserts
  product-agnosticism explicitly.
