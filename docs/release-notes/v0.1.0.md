# v0.1.0

Initial release. Kernel library for managing AI agent skills, agents, and rules
across Claude Code, OpenAI Codex, and OpenCode — a product-agnostic installer
driven by a per-product `ProductConfig`.

## Highlights

- `createCli()` factory dispatches verbs via an extensible dispatch table.
- 11 built-in CLI verbs (`install` / `uninstall` / `update` / `list` / `plan` /
  `agents` / `doctor` / `repair` / `export` / `import` / `validate`), all routed
  through `productConfig`.
- Adapter SPI v1 with built-in Claude Code / Codex / OpenCode adapters.
- Z three-layer architecture (`core` / `adapters` / `cli`) enforced by
  `architecture.test.mjs`.
- i18n seam: every user-facing string flows through `cli/strings.mjs`.
- `examples/sample-product/` — a complete worked example with bin + tests.

See `README.md` for usage.

## Notes

- Forked from the netops-agent-skills internal product at its v0.5.1; the OSS

  library and the single worked example under `examples/sample-product/` ship.
