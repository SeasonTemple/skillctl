# v0.2.0

Absorbs the product-agnostic Tier 1+2 evolution from the downstream
netops-agent-skills fork (at its terminal v0.10.0 state) into the OSS kernel.
See `docs/adr/0001`–`0005` and
`docs/plans/2026-05-18-001-feat-absorb-netops-spi-v11-and-release-discipline-plan.md`.

## Breaking Changes

For git-tag / vendor consumers (the kernel is **not** published to npm — see
`docs/adr/0005`; the npm `skillctl` name is an unrelated third-party package):

- **Removed: the `pipeline` public export and `ERR_PIPELINE_STAGE` /
  `ERR_PIPELINE_COMMIT` / `ERR_PIPELINE_PERSIST` from `index.mjs`.** The dormant
  two-tier `core/pipeline.mjs` namespace had zero internal callers (pre-delete
  consumer grep, ADR-0003) and was deleted, adopting the downstream's terminal
  single-tier installer architecture (netops ADR-0004). Migration: there are no
  known consumers; if you imported `pipeline.*`, use the `commands.*` verbs
  (`install` / `update` / `uninstall` / `repair`) which encapsulate plan +
  stage + commit inline. Pin `skillctl` to the prior tag if you need the old
  surface.
- **`repair` behavior change.** `repair` now records the staged (transformed)
  hash into `state.json` (it always re-copied from *current source*; it now
  also makes `state.json` honest about that). This fixes a latent bug where the
  next `update` after a `repair` of a `missing`/`--accept-modified` file with a
  drifted source would false-flag it as tampered and block. `repair` restores
  to **current source**, not the originally-installed snapshot.

## Added

- **Adapter SPI v1.0 → v1.1**: optional `transformAssetContent(asset, body) =>
  body` hook (identity default — existing adapters unaffected) for per-CLI
  content rewriting; new `ERR_TRANSFORM_FAILED` (public stability contract).
- `core/stage-asset.mjs` (`stageAsset` / `hashTransformed`, core-internal):
  transform + hash + stage primitives that deduplicate the three command
  staging sites and enforce the cross-stage hash invariant (`state.json`
  sha256 == `hashFile(on-disk)`) for transformed and untransformed assets.
- OpenCode adapter installs `agent` assets: `transformAssetContent` translates
  Claude-Code agent frontmatter to OpenCode's schema (frontmatter only; body
  byte-identical). Per-agent `tools:` restriction does not survive the
  translation — a named, accepted capability divergence (ADR-0002 D4).
- `scripts/lint-release-sync.mjs` (`npm run lint:release-sync`, also in
  pre-commit): asserts `package.json` version == newest
  `docs/release-notes/v*.md` by semver.
- `docs/adr/` decision-record practice (ADR-0001–0005); `CONTEXT.md`.
- Layered test-suite rebuild bound to `examples/sample-product/` (160 tests).

## Fixed

- Pre-existing `productConfig` wiring defect: `update`, `updateMulti`,
  `repair`, `uninstall`, `uninstallMulti`, `exportCommand`, and `importCommand`
  referenced `productConfig` without it being threaded through their
  signatures / `run.mjs` call sites — every one threw `ReferenceError` for any
  product. Now wired consistently with `install`.
- Crash-window hardening: `install` / `update` / `repair` now write
  `state.json` *before* promoting staged files to disk (matching `uninstall`'s
  discipline). A crash mid-operation now leaves state ahead of disk
  (recoverable: `doctor`/`repair` re-copies) instead of disk ahead of state
  (which would false-flag tamper).
