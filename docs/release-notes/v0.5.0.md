# v0.5.0

Minor: project renamed `skillctl` → `nexel`, with a new ADR
superseding ADR-0005. **No logic change and no on-disk-contract
change** — this is an identity rename plus a decision record. See
`docs/adr/0007-rename-to-nexel-and-decouple-publish-decision.md` and
`docs/plans/2026-05-18-004-refactor-rename-skillctl-to-nexel-plan.md`.

## Changed — identity rename

- **Package identity**: `package.json` `name` `skillctl` → `nexel`;
  repository / homepage / bugs URLs and the `skillctl` keyword retargeted
  to `nexel`; `package-lock.json` regenerated. `"private": true` is
  **retained** (publish remains deferred).
- **Forward-facing docs**: `README.md` / `README.zh-CN.md` / `CLAUDE.md`
  / `CONTEXT.md` / `docs/AGENT-CLI-CONTRACT.md` / `LICENSE` /
  `.husky/pre-commit` — live-identity statements renamed to `nexel`. The
  R3 mixed-file rule was applied per-statement: historical narration is
  preserved verbatim (e.g. the `CONTEXT.md` maintainer quote; the
  squatted-npm-package references keep the name `skillctl` because that
  is the third-party package's actual name), with forward pointers added
  to ADR-0007.
- **Kernel string hygiene**: the `ERR_LOCKED` lock message no longer
  embeds a product literal ("another run", brand-free). Current-state
  code comments retargeted to `nexel`; `pre-skillctl` historical-era
  comments preserved (R3).

## Added

- `docs/adr/0007-rename-to-nexel-and-decouple-publish-decision.md` —
  supersedes ADR-0005. It **verbatim-quotes and explicitly retracts**
  ADR-0005's clause "that decision starts the public-API contract clock
  and removes `private:true`", recording that the name decision is
  deliberately decoupled from the publish/contract-clock decision.
- Product-literal regression guards extended: the four divergent guards
  (`strings.test.mjs`, `help.test.mjs`, two in `prompts.test.mjs`) are
  normalized to `/\b(skillctl|netops|nexel)\b/i` so the "kernel strings
  embed no product literal" invariant survives the rename — `nexel` is
  added, the legacy `skillctl`/`netops` detectors are retained (not a
  token swap). A test-first witness asserts the guard catches
  `nexel`/`Nexel`/`NEXEL` plus the legacy literals.

## Frozen by design (R6)

- `STATE_DIRNAME` stays `.skillctl`. The on-disk state directory is a
  stability contract independent of the product mark; renaming it would
  orphan every existing `.skillctl/` directory on disk. This is the one
  deliberate place the new name does **not** propagate — recorded in
  ADR-0007. Consequently "zero production-code behavior change" is scoped
  to mean "no logic change and no on-disk-contract change".

  (`skillctl` → `nexel`) — update the name you pull. Nothing else
  changed: no behavior, no exit codes, no `--json` shapes, no on-disk
  state path (still `.skillctl`). No migration needed.
- The README npm-version badge deliberately still points at the
  unrelated third-party `skillctl` package; its disposition belongs to
  the deferred 2b npm-publish work (ADR-0007 carve-out).
- `package.json` carries `"private": true`; publication remains
  deferred while decoupled from the (now-resolved) name decision.
