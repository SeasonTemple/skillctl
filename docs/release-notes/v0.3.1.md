# v0.3.1

Patch: correctness fix to the CLI error contract. No API surface change.

## Fixed

- **Uniform `--json` error envelope for every error path.** `ERR_DIRECT_UNSUPPORTED`,
  `ERR_LOCKED`, and the generic untyped fallback in `handleError` wrote to
  stderr even in `--json` mode, contradicting the `docs/AGENT-CLI-CONTRACT.md`
  §3 contract that promises a uniform `{ ok:false, error, message, details }`
  envelope on stdout. An agent parsing stdout in `--json` mode silently lost
  these errors. All error paths now emit the uniform stdout envelope in
  `--json` mode; text-mode behavior (plugin install instructions, `DEBUG`
  stack trace) is unchanged. Surfaced as a pre-existing residual by the
  Tier-2 code review of v0.3.0 (api-contract, P2).

## Added

- `scripts/installer/cli/error-format.test.mjs` (11 cases, every error path
  in both `--json` and text mode) wired into the `test` aggregate.

## Notes

- For git-tag / vendor consumers: no public-surface change. `printHelp` /
  `renderHelp` / exports unchanged. The only behavioral change is that
  `--json` error output for the three previously-stderr paths now correctly
  lands on stdout as documented.
- `package.json` carries `"private": true`; publication remains deferred
  while the package name is provisional (ADR-0005).
