# v0.4.0

Minor: delivers the ADR-0004-deferred product-coupled test sweep. No
public API surface change. See `docs/adr/0004-...` and
`docs/plans/2026-05-18-003-feat-product-coupled-test-sweep-plan.md`.

## Added

- **Product-coupled test sweep** closing the ADR-0004 coverage gap
  (188 → 231 tests):
  - `scripts/installer/cli/commands/index.test.mjs` — 15 kernel command
    functions (`install`/`uninstall`/`update`/`repair`/`list`/`agents`/
    `doctor`/`export`+`import`/`planSelection`/`planCommandText`/
    `getRepoCommit`) via **direct in-process calls** against the
    `examples/sample-product/` fixture with real temp targets. Confirms
    the command functions ARE callable standalone with explicit
    `productConfig` + `installerVersion` (the `repair-rehash` "not
    callable standalone" header was that test's spawn-runner choice,
    not a constraint).
  - `scripts/installer/cli/prompts.test.mjs` — interactive layer
    characterization via the injectable `prompts` DI seam +
    stdout-capture for `renderBanner`.
  - `scripts/installer/cli/strings.test.mjs` — catalog-completeness +
    product-literal guard across `help`/`errors`/`run`/`help.verb`.
  - `examples/sample-product/sample-bin.test.mjs` — +6 spawn E2E
    witnesses for `run.mjs` verb handlers + `cli.mjs` composition
    (`--json` envelopes, the v0.3.1 `validate` exit-code contract, the
    v0.3.1 uniform `--json` error envelope, `--target`+multi-agent
    rejection, unknown-verb fallback).
  - `scripts/installer/core/manifest/loader.test.mjs` — +5 edge cases
    (BOM+CRLF, malformed-JSON / missing-file raw-IO contract,
    forward-compat unknown keys, real sample manifest).

## Hygiene

- Removed two **unreachable** `NetOps` dead literals — `renderBanner`'s
  default title and `gatherInstallChoices`' intro — replaced with
  product-agnostic generic constants. These functions have no
  kernel/sample caller (the interactive layer is product-bin
  territory), so this is dead-literal cleanup with **zero observable
  behavior change** for skillctl consumers — not a behavioral bug fix.
  No signature change.

## Deferred (flagged by the sweep, out of scope)

- `renderNextSteps`' hardcoded `node scripts/install-skills.mjs ...`
  bin-path literal — a stale product-specific invocation path; a
  correct replacement needs a `ProductConfig`/caller seam (architectural,
  its own plan).
- A typed-error loader layer — `loader.mjs` is intentionally pure
  IO+parse ("no validation logic"); raw `SyntaxError`/`ENOENT` are the
  validator/CLI layer's job to type. Changing that is a layering
  decision, not a test-sweep fix.

## Notes

- For git-tag / vendor consumers: no public-surface change, purely
  additive coverage. The two literal swaps are in caller-less internal
  `cli/` code; a downstream product that wraps `renderBanner` /
  `gatherInstallChoices` directly would see the new generic default.
- `package.json` carries `"private": true`; publication remains
  deferred while the package name is provisional (ADR-0005).
